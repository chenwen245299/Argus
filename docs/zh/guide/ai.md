# AI 工作流

Argus是**AI原生**的：AI不是外挂功能，不需要通过插件实现，而是贯穿整个应用的功能。用户可以通过配置自己喜欢的AI提供商和模型来使用。

## 支持的AI供应商接口格式

在**设置 → AI供应商** 中可以同时配置一个或多个提供商，Argus支持以下格式的请求:

- **OpenAI**
- **Anthropic**
- **OpenRouter**
- **Ollama**
- 任意**OpenAI 兼容**接口（自建或第三方）

用户配置时只需要在供应商的详情界面里面通过下拉框选择对应的接口格式即可。

用户可以为不同的任务配置不同的AI模型（设置 → 论文分析），同时还可以配置对应任务的提示词，更好的满足个性化需求：

<Media src="/media/1783333738705.png" caption="Argus支持为不同的任务配置不同的AI模型" />

## AI能做什么

### 元数据提取

当内置解析器无法处理时，使用AI从论文正文（或由模型）自动提取标题、作者、会议名称、年份和摘要等信息。

### 摘要提取

直接从论文原文中抽取摘要，或用 LLM 生成一段摘要（内置解析器无法获取时）。

### 论文分析

生成对论文的总结分析，用户可以在设置中配置对应的提示词和模型，针对自己的领域和工作流来调校分析。

### 单篇论文对话

与单篇论文对话。Argus会把论文文本作为上下文，用户可以让它解释、总结或答疑。对话历史会按照论文进行保存。

## 费用与 token 追踪

每次AI请求都会记录token用量和对应的费用估算（如果API返回费用，则使用API返回的费用，否则Argus根据用户配置的价格自动计算）。翻译和对话结果会内联显示所用模型、输入 / 输出 token 及费用，并在文献库文件夹里保留运行日志。
