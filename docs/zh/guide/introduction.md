# 项目简介

<Media src="media/1783340663019.png" />

**Argus** 是一款轻量、AI 原生、**本地优先**的文献阅读软件，用于阅读、分析、检索你的学术论文，并与论文对话 —— 一切都在一个桌面应用里完成。它把 PDF 与电子书阅读、AI功能、笔记、arXiv 追踪、论文关系图谱、素材库，以及全库 RAG 检索与论文分析整合到一处，让你的整个科研阅读流程集中在同一个地方且能够无缝融合AI。

> [!CAUTION]
> **本项目的大部分代码由 AI 生成 / 辅助生成。** Argus 目前功能已趋于稳定，本人已经在MacOS 26系统上高强度使用了一段时间，基本修复了大部分bug，目前在MacOS 26系统上体验流畅。Windows系统上基本功能也都能正常运行。

## 设计理念

- **本地优先** —— 你的 PDF、笔记、高亮、对话记录和 AI 分析，都存放在**你自己**选择的文献库文件夹里，就在**你自己**的电脑上。不绑定云、无需注册账号、数据能够轻松迁移。
- **AI 原生** —— AI 不是外挂功能，也不需要通过额外的插件引入，而是直接将AI融合进APP之中，作为原生能力，用户可以自由的使用AI来分析、讲解、总结论文，以及回答各种问题。AI供应商支持各种常见的接口格式（OpenAI、Anthropic、OpenRouter、Ollama，或任意 OpenAI 兼容接口）
- **轻量** —— 基于 [Tauri](https://tauri.app/) 构建，安装体积小（20MB~）、启动快。

## 你能用它做什么

| 领域 | 简介 |
| --- | --- |
| [文献库管理](/zh/guide/library) | 导入 PDF 与电子书，用文件夹和标签整理，记录阅读状态，集中维护元数据。 |
| [阅读与笔记](/zh/guide/reading) | 标签页阅读 PDF、高亮批注，并用带 OCR 兜底的全文提取撰写富文本笔记。 |
| [AI 工作流](/zh/guide/ai) | 配置任意提供商与模型，提取元数据与摘要、生成可配置分析，并与任意论文对话。 |
| [画布](/zh/guide/canvas) | 把论文作为节点排布、连接相关工作，梳理发展脉络，并导出关系图谱。 |
| [素材库](/zh/guide/snippets) | 收集阅读时的摘录，基于独立向量库做语义检索，方便写作时查找与引用。 |
| [向量图谱](/zh/guide/embedding-map) | 把文献库的 embedding 投影到二维图，可视化论文聚类、发现潜在关联。 |
| [arXiv / bioRxiv 追踪](/zh/guide/arxiv) | 定时抓取预印本、按 AI 相关性过滤，在每日收件箱集中处理。 |
| [RAG问答](/zh/guide/rag) | 构建本地向量索引，带来源地对整个文献库提问与检索。 |

## 适合谁

Argus 主要面向**科研人员、博士生，以及任何需要管理不断增长的文献库和写论文**的人 —— 希望用一个轻量化的AI原生的软件来辅助阅读、分析和整理文献的人。
